# Feature Suggestions

At the start of each new session or when the user asks "what should I work on next", read `Docs/ToDo.txt` and suggest the next 2-3 features to implement based on these priorities:

1. **Stability first** — if there are known bugs or untested code (e.g. character generation hasn't been playtested yet), suggest fixing/testing those before new features.
2. **Foundation before content** — prefer systems that other features depend on (e.g. NPC system before storyline, Lua AI before complex enemy behaviour).
3. **Small wins** — if the user has limited credits, suggest quick-to-implement items that add visible gameplay value.
4. **Balance variety** — alternate between combat, exploration, UI, and content tasks to keep the game feeling more complete each session.

When suggesting, briefly explain WHY each feature is a good next step (what it unblocks or adds to the player experience).
