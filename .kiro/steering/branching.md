# Branching

All development work must happen on feature branches — never commit directly to `main`.

- Before starting work, check which branch you're on. If on `main`, create or switch to a feature branch.
- Branch naming: use kebab-case matching the spec name (e.g. `equipment-system`, `hit-chance-system`)
- Push to the feature branch, not main.
- `main` is updated only via pull request merges.
